# 시큐어 코딩 규칙 (SFR:10.1.1)

제품 개발 시 소스코드에 보안약점이 존재하지 않도록 아래 규칙을 적용한다. (선택 항목)

## 적용 규칙
- **입력 검증**: 모든 외부 입력(웹·RTSP·연동 API)은 길이·형식·범위를 검증한다.
- **메모리 안전**: C 코드는 경계 검사, `strcpy`/`sprintf` 등 위험 함수 금지(`strncpy`/`snprintf` 사용).
- **암호 직접 호출 금지**: 모든 암호 연산은 `crypto_hal` 인터페이스만 통과(금지 알고리즘 원천 차단).
- **비밀정보**: 하드코딩 금지(키·비밀번호는 프로비저닝 주입), 로그·화면에 미출력(SFR:2.5.1).
- **오류 처리**: 실패 사유를 사용자에게 노출하지 않음(SFR:2.5.2), 내부 감사로그에만 상세 기록.

## 점검 방법
- 정적분석 도구(예: `cppcheck`, `clang-tidy`)를 CI에 통합.
- Buildroot `BR2_ENABLE_DEBUG=n`, 컴파일러 하드닝 플래그(`-D_FORTIFY_SOURCE=2 -fstack-protector-strong`) 적용.
- 코드 리뷰 시 위 규칙 준수 여부 확인.
